# 스포츠센터 회원 관리 프로그램 - 기능 명세서 (Functional Specification)

본 문서는 Java Swing 및 MariaDB를 기반으로 개발되는 '스포츠센터 회원 관리 프로그램'의 핵심 기능 및 세부 구현 요구사항을 명세합니다.

## 1. 개요

본 프로그램은 데스크톱 환경에서 스포츠센터 직원이 회원 정보를 직관적이고 효율적으로 관리(CRUD)할 수 있도록 지원하는 클라이언트 애플리케이션입니다.

## 2. 주요 기능 명세

### 2.1. 회원 관리 기능 (Member Management)

가장 핵심이 되는 기능으로, 스포츠센터 회원의 기본 인적 정보를 데이터베이스에 저장하고 관리합니다.

- **회원 등록 (Create)**
  - **입력 항목:** 이름(필수), 연락처(필수), 성별, 생년월일, 비고
  - **유효성 검사:** 이름 및 연락처 누락 시 경고창(`JOptionPane`) 표시.
  - **기능:** 신규 회원 등록만 전담하며, 개인정보를 MariaDB의 `members` 테이블에 `INSERT` 합니다.
- **회원 목록 조회 (Read)**
  - **전체 조회:** 프로그램 실행 및 시작 화면 로드 시 DB에 저장된 전체 회원 목록을 `JTable`에 출력합니다.
  - **검색 기능:** '이름' 또는 '연락처'의 일부만 입력해도 일치하는 회원을 검색(`SELECT ... LIKE`)하여 테이블에 갱신합니다.
- **회원 상세 조회 및 정보 수정 (Read & Update)**
  - **상세 정보 페이지:** `JTable`의 '상세' 버튼을 클릭하면, 해당 회원의 '상세 정보 페이지'로 이동합니다.
  - 상단에는 회원의 기본 정보가 표시되고, 하단에는 해당 회원의 수강 내역이 표출됩니다.
  - 상세 정보 페이지 상단의 **[정보 수정]** 버튼 클릭 시, 기존 정보가 채워진 팝업 창이 열리며 정보를 수정할 수 있습니다. 수정한 내용을 저장하면 DB의 해당 레코드를 `UPDATE` 하고 상세 페이지를 새로고침합니다.
  - **수강 취소 기능:** 하단 수강 내역 목록(`JTable`)에서 특정 강좌를 선택하고 **[수강 취소]** 버튼을 클릭하면, 확인 다이얼로그를 거친 후 DB의 `enrollments` 테이블에서 해당 레코드를 삭제(`DELETE`)합니다.
- **회원 삭제 (Delete)**
  - **기능:** 삭제할 회원을 선택하고 삭제를 진행합니다.
  - **확인 절차:** 실수로 인한 삭제를 방지하기 위해 "정말 삭제하시겠습니까?"라는 확인 다이얼로그(`JOptionPane.showConfirmDialog`)를 띄웁니다.
  - 확인 시 DB에서 해당 레코드를 `DELETE` 하고 목록을 새로고침합니다.

### 2.2. 수강 관리 기능 (Enrollment Management)

회원이 센터에서 운영 중인 강좌를 수강할 수 있도록 등록 및 관리하는 기능입니다.

- **수강 등록 (Create)**
  - **접근 방법:** 회원 '상세 정보 페이지' 내에 있는 **[수강 등록]** 버튼을 통해 수강 등록 팝업 창을 엽니다.
  - **기능:** DB의 `lessons` 테이블에 존재하는 강좌를 선택하여 특정 회원에게 수강 등록을 수행합니다. 등록된 수강 내역은 `enrollments` 테이블에 저장됩니다.
  - **수강 기간 자동 설정:**
    - 해당 회원이 당월 등록한 이력이 없는 경우: 시작일은 오늘 날짜, 종료일은 당월 말일로 자동 설정됩니다.
    - 해당 회원이 이미 당월 수강 중인 경우: 익월 등록으로 자동 간주되어 시작일은 익월 1일, 종료일은 익월 말일로 설정됩니다.
  - **기간 변경:** 자동 설정된 시작일 및 종료일은 수강 등록 창에서 직원이 필요에 따라 임의로 변경할 수 있습니다.

### 2.3. 강좌 관리 기능 (Lesson Management)

센터에서 운영하는 강좌 정보를 관리합니다.

- **강좌 관리 (CRUD)**: 새로운 강좌 정보를 등록, 조회, 수정, 삭제하며, 해당 데이터는 `lessons` 테이블에 저장됩니다.
- **유효성 검사 (안전 장치):** 강좌 수정 및 삭제 시 해당 강좌를 수강 중인 회원이 한 명이라도 존재할 경우, 경고 메시지를 표시하고 수정/삭제를 막아야 합니다.

### 2.4. 부가 기능 (만료 회원 관리 및 필터링)

- **만료일 도래 회원 강조:** 회원 목록 조회 시 멤버십 종료일이 현재 날짜 기준 7일 이내이거나 이미 만료된 회원의 경우, **강좌가 만료되었음에도 불구하고 익월 등록 기록이 없다면** `JTable` 목록에서 행의 배경색이나 텍스트 색상을 다르게 표시하여(예: 붉은색) 직원이 쉽게 인지할 수 있도록 합니다.
- **필터링 보기:** '전체 회원', '활성 회원', '만료 회원'을 라디오 버튼이나 콤보박스로 선택하여 목록을 필터링할 수 있는 기능을 제공합니다.

### 2.5. UI / UX 및 예외 처리

- **비동기 UI 처리:** 데이터베이스 조회, 삽입, 수정, 삭제 작업 시 `SwingWorker`나 별도의 스레드를 사용하여 메인 UI(EDT)가 멈추는(Freezing) 현상을 방지합니다.
- **초기화 (Reset):** 입력 폼에 작성 중이던 모든 내용을 지우고 빈 상태로 돌리는 '초기화' 버튼을 제공합니다.
- **에러 핸들링:** DB 연결 실패, 쿼리 오류, 타임아웃 발생 시 콘솔 로그뿐만 아니라 사용자에게 명확한 에러 메시지 팝업을 띄워 상황을 인지시킵니다.

## 3. 화면(View) 구성 레이아웃 (안)

- **최상단 버튼 영역 (네비게이션):** 화면 최상단에는 **[뒤로가기]**, **[회원 관리(초기화면)]**, **[회원 등록]**, **[센터 관리]** 버튼이 배치되어 원활한 화면 전환 및 복귀 기능을 제공합니다.
- **시작 화면 (회원 목록):** 로그인(해당 시) 또는 프로그램 실행 직후 화면은 DB에서 불러온 회원 목록을 보여주는 데이터 그리드(`JTable`) 화면으로 고정됩니다.
  - **목록 열 구성:** `id` / `이름` / `마지막 수강 강좌명` / `수강 시작일 ~ 만료일` / `가입일` / `상세(버튼)` 데이터를 표시합니다.
- **회원 상세 정보 페이지:**
  - 회원 목록에서 `상세` 버튼 클릭 시 해당 회원의 상세 정보 페이지로 이동합니다.
  - **화면 구성:** 상단에는 회원 등록 시 작성한 기본 정보와 **[정보 수정]**, **[수강 등록]** 버튼이 위치하며, 하단에는 해당 회원의 수강 내역이 표출됩니다.
- **팝업 창(Dialog) 기반 동작:**
  - [회원 등록] 버튼 클릭 시 신규 회원 정보만 입력받을 수 있는 팝업 창이 표시됩니다.
  - 상세 정보 페이지의 [정보 수정] 버튼 클릭 시, 정보가 채워진 상태로 수정 가능한 팝업 창이 표시됩니다.
  - 상세 정보 페이지의 [수강 등록] 버튼 클릭 시, 회원 및 강좌를 선택하고 기간을 설정할 수 있는 수강 등록 팝업 창이 표시됩니다.
- **센터 관리 화면 전환:**
  - 최상단의 [센터 관리] 버튼 클릭 시 팝업이 아닌 현재 화면 전체가 '센터 관리 화면'으로 변경됩니다.
  - 센터 관리 화면 내에 위치한 **[강좌 관리]** 버튼을 클릭하면, 강좌를 등록/수정/삭제할 수 있는 팝업 창이 표시됩니다.

## 4. 데이터베이스 테이블 구조 (Reference)

기능 구현을 위해 최소 다음과 같은 테이블과 필드가 구성되어야 합니다.

### `members` (회원 테이블)

- `id` (PK, Auto Increment): 회원 고유 번호
- `name` (VARCHAR): 회원 이름
- `phone` (VARCHAR): 연락처 (예: 010-1234-5678)
- `gender` (VARCHAR or ENUM): 성별
- `birth_date` (DATE): 생년월일
- `created_at` (DATE or TIMESTAMP): 가입일
- `notes` (TEXT): 기타 특이사항

### `lessons` (강좌 테이블)

- `id` (PK, Auto Increment): 강좌 고유 번호
- `name` (VARCHAR): 강좌 이름
- `day_of_week` (VARCHAR): 강좌 요일
- `time` (VARCHAR): 강좌 시간
- `instructor_name` (VARCHAR): 강사명
- `capacity` (INT): 강좌 정원
- `price` (INT): 강좌 가격

### `enrollments` (수강 내역 테이블)

- `id` (PK, Auto Increment): 수강 고유 번호
- `member_id` (FK): 회원 고유 번호 (`members.id` 참조)
- `lesson_id` (FK): 강좌 고유 번호 (`lessons.id` 참조)
- `start_date` (DATE): 수강 시작 일자
- `end_date` (DATE): 수강 만료 일자
- `payment_date` (DATE): 결제일

---

**※ 제약 사항:**
본 애플리케이션은 엄격한 MVC 패턴을 준수하며, 위 기능들을 구현할 때 View 계층에서는 절대 DB 통신 로직이 포함되어서는 안 됩니다.
