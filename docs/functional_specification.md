# 스포츠센터 회원 관리 프로그램 - 기능 명세서 (Functional Specification)

본 문서는 Java Swing 및 MariaDB를 기반으로 개발되는 '스포츠센터 회원 관리 프로그램'의 핵심 기능 및 세부 구현 요구사항을 명세합니다.

## 1. 개요
본 프로그램은 데스크톱 환경에서 스포츠센터 직원이 회원 정보를 직관적이고 효율적으로 관리(CRUD)할 수 있도록 지원하는 클라이언트 애플리케이션입니다.

## 2. 주요 기능 명세

### 2.1. 회원 관리 기능 (Member Management - CRUD)
가장 핵심이 되는 기능으로, 스포츠센터 회원의 정보를 데이터베이스에 저장하고 관리합니다.

*   **회원 등록 (Create)**
    *   **입력 항목:** 이름(필수), 연락처(필수), 성별, 생년월일, 멤버십 시작일, 멤버십 종료일, 비고
    *   **유효성 검사:** 이름 및 연락처 누락 시 경고창(`JOptionPane`) 표시, 날짜 형식 검증.
    *   **기능:** 입력된 정보를 MariaDB의 `members` 테이블에 `INSERT` 합니다.
*   **회원 목록 조회 (Read)**
    *   **전체 조회:** 프로그램 실행 및 초기 화면 로드 시 DB에 저장된 전체 회원 목록을 `JTable`에 출력합니다.
    *   **검색 기능:** '이름' 또는 '연락처'의 일부만 입력해도 일치하는 회원을 검색(`SELECT ... LIKE`)하여 테이블에 갱신합니다.
*   **회원 정보 수정 (Update)**
    *   **기능:** `JTable`에서 특정 회원을 클릭(선택)하면, 해당 회원의 상세 정보가 입력 폼에 바인딩됩니다.
    *   사용자가 정보를 수정한 후 '수정' 버튼을 누르면 DB의 해당 레코드를 `UPDATE` 하고 목록을 새로고침합니다.
*   **회원 삭제 (Delete)**
    *   **기능:** `JTable`에서 삭제할 회원을 선택하고 '삭제' 버튼을 누릅니다.
    *   **확인 절차:** 실수로 인한 삭제를 방지하기 위해 "정말 삭제하시겠습니까?"라는 확인 다이얼로그(`JOptionPane.showConfirmDialog`)를 띄웁니다.
    *   확인 시 DB에서 해당 레코드를 `DELETE` 하고 목록을 새로고침합니다.

### 2.2. 만료 회원 관리 및 필터링 기능 (부가 기능)
*   **만료일 도래 회원 강조:** 멤버십 종료일이 현재 날짜 기준 7일 이내이거나 이미 만료된 회원은 `JTable` 목록에서 행의 배경색이나 텍스트 색상을 다르게 표시하여(예: 붉은색) 직원이 쉽게 인지할 수 있도록 합니다.
*   **필터링 보기:** '전체 회원', '활성 회원', '만료 회원'을 라디오 버튼이나 콤보박스로 선택하여 목록을 필터링할 수 있는 기능을 제공합니다.

### 2.3. UI / UX 및 예외 처리
*   **비동기 UI 처리:** 데이터베이스 조회, 삽입, 수정, 삭제 작업 시 `SwingWorker`나 별도의 스레드를 사용하여 메인 UI(EDT)가 멈추는(Freezing) 현상을 방지합니다.
*   **초기화 (Reset):** 입력 폼에 작성 중이던 모든 내용을 지우고 빈 상태로 돌리는 '초기화' 버튼을 제공합니다.
*   **에러 핸들링:** DB 연결 실패, 쿼리 오류, 타임아웃 발생 시 콘솔 로그뿐만 아니라 사용자에게 명확한 에러 메시지 팝업을 띄워 상황을 인지시킵니다.

## 3. 화면(View) 구성 레이아웃 (안)
화면은 크게 세 영역으로 나뉩니다.
1.  **상단 영역 (검색 및 필터):** 검색어 입력 필드, 검색 버튼, 필터링 옵션
2.  **좌측/하단 영역 (회원 정보 입력 폼):** 이름, 연락처, 만료일 등을 입력하고 [등록], [수정], [삭제], [초기화] 버튼이 위치하는 패널
3.  **우측/중앙 영역 (회원 데이터 그리드):** DB에서 불러온 회원 목록을 보여주는 `JTable` 및 스크롤 패널(`JScrollPane`)

## 4. 데이터베이스 테이블 구조 (Reference)
기능 구현을 위해 최소 다음과 같은 필드가 `members` 테이블에 포함되어야 합니다.
*   `id` (PK, Auto Increment): 회원 고유 번호
*   `name` (VARCHAR): 회원 이름
*   `phone` (VARCHAR): 연락처 (예: 010-1234-5678)
*   `gender` (VARCHAR or ENUM): 성별
*   `birth_date` (DATE): 생년월일
*   `start_date` (DATE): 멤버십 시작 일자
*   `end_date` (DATE): 멤버십 만료 일자
*   `notes` (TEXT): 기타 특이사항

---
**※ 제약 사항:**
본 애플리케이션은 엄격한 MVC 패턴을 준수하며, 위 기능들을 구현할 때 View 계층에서는 절대 DB 통신 로직이 포함되어서는 안 됩니다.
