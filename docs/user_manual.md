# 스포츠센터 회원 관리 프로그램 - 사용설명서

이 문서는 스포츠센터 회원 관리 애플리케이션의 핵심 사용법을 안내하는 공식 사용자 매뉴얼입니다.

---

## 1. 프로그램 개요

본 애플리케이션은 스포츠센터의 **회원**, **강좌**, 그리고 회원의 **수강 내역**을 통합하여 관리할 수 있는 직관적인 데스크톱 기반 소프트웨어입니다.
모든 데이터는 MariaDB를 통해 실시간으로 저장 및 관리되며, 백그라운드 비동기 처리를 통해 쾌적한 사용 환경을 제공합니다.

---

## 2. 메인 화면 구성

프로그램을 실행하면 다음과 같은 메인 화면이 표시됩니다.

- **상단 컨트롤 패널**: 회원을 검색할 수 있는 텍스트 필드와 `[검색]`, `[초기화]`, `[회원 추가]`, `[강좌 추가]` 등의 주요 액션 버튼이 배치되어 있습니다.
- **좌측 테이블 (회원 목록)**: 현재 등록된 모든 회원의 정보(이름, 연락처, 성별, 생년월일 등)가 한눈에 나타납니다.
- **우측 테이블 (강좌 목록)**: 센터에서 운영 중인 전체 강좌 정보(강좌명, 요일, 시간, 강사명, 정원, 가격)가 나타납니다.

---

## 3. 회원 관리 기능 가이드

### 3.1. 회원 추가
1. 메인 화면 상단의 `[회원 추가]` 버튼을 클릭합니다.
2. 새 회원 팝업 창이 열리면 이름, 연락처, 성별, 생년월일, 비고 등 정보를 입력합니다.
   * `이름`과 `연락처`는 필수 입력 항목입니다.
3. `[저장]` 버튼을 누르면 성공적으로 데이터베이스에 등록되며 목록에 즉시 반영됩니다.

### 3.2. 회원 검색
1. 상단 중앙의 검색바에 찾고자 하는 회원의 **이름** 또는 **연락처** 일부를 입력합니다.
2. `[검색]` 버튼을 클릭하면 조건에 맞는 회원만 목록에 필터링되어 출력됩니다.
3. 검색을 취소하려면 `[초기화]` 버튼을 누르세요.

### 3.3. 회원 상세 보기 및 수정, 삭제
1. 좌측 회원 목록 테이블에서 원하는 회원을 **더블클릭**합니다.
2. 회원 상세 정보 창이 열리며, 이 창에서 정보의 열람 및 수정이 가능합니다.
3. 정보 변경 후 하단의 `[수정]` 버튼을 클릭하여 덮어씁니다.
4. 삭제가 필요한 경우 하단의 `[삭제]` 버튼을 클릭한 후 확인창에 동의하여 삭제를 완료합니다.

---

## 4. 강좌 관리 기능 가이드

### 4.1. 강좌 개설
1. 상단 우측의 `[강좌 추가]` 버튼을 클릭합니다.
2. 강좌명, 요일, 시간, 강사명, 정원, 가격을 입력하고 `[저장]`을 클릭합니다.

### 4.2. 강좌 정보 수정 및 폐강
1. 우측 강좌 테이블에서 관리하고자 하는 강좌를 **더블클릭**합니다.
2. 팝업 창에서 정원이나 가격 등을 수정한 후 `[수정]` 버튼을 클릭합니다.
3. 더 이상 강좌를 운영하지 않는다면 `[삭제]`를 클릭하여 지울 수 있습니다.

---

## 5. 수강 관리 기능 가이드

회원이 특정 강좌를 듣도록 등록하고 관리하는 방법입니다.

### 5.1. 수강 신청
1. 메인 화면 좌측에서 회원을 **더블클릭**하여 "회원 상세 창"을 엽니다.
2. 상세 창 우측 하단부의 `[수강 신청]` 버튼을 클릭합니다.
3. 수강 신청 팝업에서 현재 운영 중인 강좌를 콤보박스로 선택하고 수강 시작일 및 종료일을 지정합니다.
4. `[저장]`을 클릭하면 하단 수강 내역 목록에 즉시 추가됩니다.
   * **주의사항**: 이미 회원이 해당 강좌를 수강 중이거나, 강좌의 현재 수강 인원이 정원(Capacity)을 가득 채웠다면 신청이 거부됩니다.

### 5.2. 수강 취소
1. 회원 상세 창 하단의 "수강 내역 테이블"에서 취소하려는 행을 클릭(선택)합니다.
2. 테이블 밑의 `[수강 취소]` 버튼을 클릭하여 수강을 종료합니다.

---

## 6. 문제 해결 및 팁

- **프로그램이 멈춘 것처럼 보일 때**: 데이터베이스 응답이 지연되는 경우 화면이 잠시 멈출 수 있으나 프로그램 내부에 비동기 스레드가 적용되어 있어 잠시 기다리시면 정상 처리됩니다.
- **DB 오류 발생**: 프로그램 실행 직후 에러 팝업이 발생하면, 환경 변수(`.env` 파일)의 아이디/비밀번호 정보와 MariaDB 도커 컨테이너가 정상 구동 중인지 확인해 주시기 바랍니다.
