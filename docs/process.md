# 작업 진행 상황 (Process)

## 대기 중인 작업: DB 비밀번호 분리 (.env 방식 적용)
사용자님의 결정에 따라 3번 방식인 `.env` 파일과 `dotenv-java` 라이브러리를 활용하여 DB 하드코딩 패스워드를 안전하게 분리하는 작업을 대기 중입니다.

### 📝 실행 예정 작업 상세 내역
1. **`pom.xml` 수정:** 
   - 환경 변수 파일을 쉽게 읽어들일 수 있도록 `<dependencies>` 내부에 `io.github.cdimascio:dotenv-java:3.0.0` 라이브러리를 추가합니다.
2. **프로젝트 최상단 루트에 `.env` 파일 생성:** 
   - `DB_USER=root`, `DB_PASSWORD=5058` 형태의 데이터를 작성하여 설정값을 코드에서 완전히 분리합니다.
3. **`.gitignore` 파일 업데이트:** 
   - 새롭게 생성한 `.env` 파일이 실수로라도 GitHub 리포지토리에 푸시되지 않도록(보안 위협 방지), 파일 목록의 가장 밑에 `.env`를 등록합니다.
4. **`DBConnectionUtil.java` 코드 리팩토링:** 
   - `private static final String PASSWORD = "5058";` 형태의 하드코딩을 삭제합니다.
   - `Dotenv.load().get("DB_PASSWORD")` 메서드를 활용하여 동적으로 패스워드와 유저 정보를 가져오도록 클래스를 안전하게 리팩토링합니다.
